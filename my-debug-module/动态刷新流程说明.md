## 核心刷新方法：`DtpRegistry.doRefresh()`

## 刷新流程详解

### 1. 入口方法：`DtpRegistry.refresh()`


### 2. 核心刷新方法：`DtpRegistry.doRefresh()`

**这是实际将新属性刷新到线程池的地方！**

```java
// 第 151-198 行
public static void doRefresh(DtpExecutor dtpExecutor, ThreadPoolProperties properties) {
    
    // ========== 刷新最大线程数 ==========
    if (properties.getMaximumPoolSize() >= 0) {
        // 调用 ThreadPoolExecutor.setMaximumPoolSize()
        dtpExecutor.setMaximumPoolSize(properties.getMaximumPoolSize());
    }
    
    // ========== 刷新核心线程数 ==========
    // jdk1.8 bug：setCorePoolSize未与maximumPoolSize比较，所以这里手动检查
    if (properties.getCorePoolSize() > dtpExecutor.getMaximumPoolSize()) {
        throw new IllegalArgumentException();
    }
    
    if (properties.getCorePoolSize() >= 0) {
        // 调用 ThreadPoolExecutor.setCorePoolSize()
        dtpExecutor.setCorePoolSize(properties.getCorePoolSize());
    }
    
    // ========== 刷新线程存活时间 ==========
    if (properties.getKeepAliveTime() > 0 && properties.getUnit() != null) {
        // 调用 ThreadPoolExecutor.setKeepAliveTime()
        dtpExecutor.setKeepAliveTime(properties.getKeepAliveTime(), properties.getUnit());
    }
    
    // ========== 刷新核心线程超时设置 ==========
    dtpExecutor.allowCoreThreadTimeOut(properties.isAllowCoreThreadTimeOut());
    
    // ========== 刷新拒绝策略 ==========
    String originRejectedName = dtpExecutor.getRejectHandlerName();
    if (StringUtils.isNotBlank(properties.getRejectedHandlerType()) &&
            !originRejectedName.contains(properties.getRejectedHandlerType())) {
        // 调用 ThreadPoolExecutor.setRejectedExecutionHandler()
        dtpExecutor.setRejectedExecutionHandler(RejectHandlerGetter.getProxy(properties.getRejectedHandlerType()));
        dtpExecutor.setRejectHandlerName(properties.getRejectedHandlerType());
    }
    
    // ========== 刷新队列容量 ==========
    if (properties.getQueueCapacity() > 0 &&
            Objects.equals(properties.getQueueType(), QueueTypeEnum.VARIABLE_LINKED_BLOCKING_QUEUE.getName())) {
        
        BlockingQueue<Runnable> blockingQueue = dtpExecutor.getQueue();
        if (blockingQueue instanceof VariableLinkedBlockingQueue) {
            // 调用 VariableLinkedBlockingQueue.setCapacity()
            ((VariableLinkedBlockingQueue<Runnable>) blockingQueue).setCapacity(properties.getQueueCapacity());
        }
    }
    
    // ========== 刷新通知项 ==========
    if (CollUtil.isEmpty(properties.getNotifyItems())) {
        dtpExecutor.setNotifyItems(getDefaultNotifyItems());
        return;
    }
    NotifyHelper.setExecutorNotifyItems(dtpExecutor, dtpProperties, properties);
    dtpExecutor.setNotifyItems(properties.getNotifyItems());
}
```

## 关键点说明

### 1. `DtpExecutor` 继承关系

```java
public class DtpExecutor extends ThreadPoolExecutor {
    // ...
}
```

**重要**：`DtpExecutor` 继承自 `ThreadPoolExecutor`，所以：
- `setCorePoolSize()` → 调用的是 **`ThreadPoolExecutor.setCorePoolSize()`**
- `setMaximumPoolSize()` → 调用的是 **`ThreadPoolExecutor.setMaximumPoolSize()`**
- `setKeepAliveTime()` → 调用的是 **`ThreadPoolExecutor.setKeepAliveTime()`**
- `setRejectedExecutionHandler()` → 调用的是 **`ThreadPoolExecutor.setRejectedExecutionHandler()`**

这些方法会**直接修改 `ThreadPoolExecutor` 的内部状态**，立即生效！

### 3. 完整调用链

```
配置中心（Nacos/Apollo/Zookeeper）配置变更
    ↓
监听器监听到变化（如：NacosRefresher.onChange()）
    ↓
调用 AbstractRefresher.refresh()
    ↓
调用 DtpRegistry.refresh(DtpProperties)
    ↓
遍历所有线程池，调用 DtpRegistry.refresh(DtpExecutor, ThreadPoolProperties)
    ↓
【核心】调用 DtpRegistry.doRefresh(DtpExecutor, ThreadPoolProperties)
    ↓
调用 ThreadPoolExecutor.setCorePoolSize()      ← 刷新核心线程数
调用 ThreadPoolExecutor.setMaximumPoolSize()    ← 刷新最大线程数
调用 ThreadPoolExecutor.setKeepAliveTime()      ← 刷新线程存活时间
调用 ThreadPoolExecutor.setRejectedExecutionHandler() ← 刷新拒绝策略
调用 VariableLinkedBlockingQueue.setCapacity()  ← 刷新队列容量
    ↓
参数立即生效，影响后续任务执行
```

## 注意事项

1. **JDK 1.8 Bug**：`setCorePoolSize()` 不会自动与 `maximumPoolSize` 比较，所以代码中手动检查（第 158-160 行）

2. **队列容量限制**：只有 `VariableLinkedBlockingQueue` 类型可以动态调整容量，其他队列类型不支持

3. **立即生效**：所有调整都是**立即生效**的，无需重启应用

4. **线程安全**：`ThreadPoolExecutor` 的这些 setter 方法都是线程安全的
